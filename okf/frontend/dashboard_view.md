# Dashboard View

The `Dashboard.js` component is the main screen after onboarding. ~845 lines.

## Layout

- **Top bar**: Title "Tasks", filter pills (Priority: All/Critical/High | Status | Type: Work/Personal), Discover and Refresh buttons
- **Stats bar**: Total tasks, active, completed, high priority count
- **Task cards**: Each card shows title (with group name subtitle), importance stars, status, context preview, latest progress, assignee avatar (initials), message count, action buttons (deep-dive, status change, nudge)
- **Collapsible task detail modal**: Shows full task info, status change pills (Active/Completed/Archived), nudge generation

## Card Design

- **Left accent bar**: 4px colored bar (purple IMP 5, orange IMP 4, blue IMP 3, transparent IMP 1-2)
- **Background tint**: Subtle gradient tint based on importance
- **Importance display**: Star emojis (⭐/⭐⭐/⭐⭐⭐)
- **Checkbox**: Neutral border, green fill on check, triggers completion animation
- **Completion animation**: 280ms fade+slide + checkPop keyframe
- **Hover**: Simple background change to `var(--surface-container)`

## Pipeline Controls

- **Discover Tasks button**: Calls `POST /api/pipeline/discover`. Shows a modal with SSE progress events per group.
- **Refresh button**: Calls `POST /api/pipeline/refresh`. Same streaming modal.
- **Deep-dive**: Each task card has a "Deep Dive" button that calls `POST /api/pipeline/deep-dive`. Opens the DeepDive drawer.

## Nudge Panel

Each task has a nudge button that calls `api.generateNudges(taskId)`. Shows 3 options (gentle, passive-aggressive, aggressive) generated by Gemini. Sends via v1 `/api/send` endpoint.

## SSE Streaming

During discover/refresh, a modal displays real-time per-group progress. Uses `discoverTasksStream()` and `refreshTasksStream()` from the API client.
